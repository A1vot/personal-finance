# План развития проекта (Roadmap)

## 1. Краткое описание

Roadmap определяет направления развития приложения и будущие улучшения. Цели распределены по этапам: ближайшие задачи, улучшения среднего уровня и долгосрочные планы.

## 2. Ближайшие задачи (MVP+)

- **Пользовательские категории**. Возможность создавать свои категории доходов и расходов.
- **Редактирование транзакций**. Возможность изменять сумму, дату, категорию и описание.
- **Улучшение валидации форм**. Проверка отрицательных сумм, дат в будущем и т. д.

## 3. Улучшения среднего уровня

- **Графики и визуализация**. Диаграммы расходов/доходов по месяцам (Chart.js).
- **Экспорт данных**. Экспорт транзакций в CSV/Excel.
- **Импорт данных**. Загрузка CSV с транзакциями.
- **Расширенная фильтрация**. Фильтр по категории, сумме, типу операции.
- **Пагинация транзакций**. Удобная навигация при большом количестве записей.

## 4. Долгосрочные планы

- **Мобильная адаптация интерфейса**. Оптимизация Dashboard под смартфоны.
- **Теги для транзакций**. Дополнительная классификация операций.
- **Многоязычность**. Поддержка нескольких языков (RU/EN/DE).
- **Уведомления и напоминания**. Напоминания о превышении бюджета или регулярных платежах.
- **API (REST)**. Открытый интерфейс для мобильных приложений и интеграций.

## 5. Возможные будущие направления

- **Поддержка нескольких бюджетов**. Например: личный, семейный, рабочий.
- **Поддержка нескольких валют**. Автоматическая конвертация по курсу.
- **Интеграция с банками**. Импорт транзакций из банковских выписок.

## 6. Итог

Roadmap определяет стратегию развития приложения:
- ближайшие задачи — улучшение UX и базовых функций
- среднесрочные — аналитика, экспорт, фильтры
- долгосрочные — масштабирование, API, мобильность

Проект имеет чёткую траекторию роста и может развиваться как полноценный финансовый инструмент.
