# Модели данных (Data Model)

## 1. Общее описание

Проект использует три основные модели:
- **Budget** — месячный бюджет пользователя
- **Category** — категории доходов и расходов
- **Transaction** — финансовые операции

Связи между моделями:

```
User 1 — 1 Budget
User 1 — * Transaction
Category 1 — * Transaction
```

## 2. Модель Budget

### Назначение

Хранит месячный лимит бюджета, привязанный к конкретному пользователю.

### Поля

|Поле|Тип|Описание|
|---|---|---|
|**user**|OneToOne(User)|Пользователь, владелец бюджета|
|**monthly_limit**|Decimal(10,2)|Месячный лимит расходов|

### Особенности

- Один пользователь → один бюджет
- При удалении пользователя бюджет удаляется (`CASCADE`)
- Строковое представление: _«Бюджет username»_
- Используется в Dashboard для расчёта остатка

## 3. Модель Category

### Назначение

Категории доходов и расходов, используемые при создании транзакций.

### Поля

|Поле|Тип|Описание|
|---|---|---|
|**name**|CharField(100)|Название категории (уникальное)|
|**type**|CharField(choices)|Тип категории: доход или расход|

### Особенности

- Уникальные названия категорий
- Типы: `"income"` или `"expense"`
- Строковое представление: _«Название (Тип)»_
- Сортировка: доходы → расходы, внутри — по имени
- Используется в отчётах Dashboard
    

## 4. Модель Transaction

### Назначение

Финансовая операция пользователя: доход или расход.

### Поля

|Поле|Тип|Описание|
|---|---|---|
|**user**|ForeignKey(User)|Владелец транзакции|
|**category**|ForeignKey(Category)|Категория операции|
|**amount**|Decimal(10,2)|Сумма операции|
|**date**|DateField|Дата транзакции|
|**description**|CharField(255)|Описание (необязательно)|

### Особенности

- При удалении категории транзакции не удаляются (`PROTECT`)
- При удалении пользователя транзакции удаляются (`CASCADE`)
- Сортировка: новые транзакции первыми
- Строковое представление: _«Категория: сумма от дата»_
- Используется в фильтрах, отчётах и расчёте бюджета

## 5. Диаграмма связей (Crow’s Foot)

![alt text](<Crow’s Foot.png>)

### Объяснение связей

- **User → Budget**: один пользователь имеет один бюджет
- **User → Transaction**: один пользователь имеет много транзакций
- **Category → Transaction**: одна категория используется в нескольких транзакциях

## 6. Бизнес‑правила

- У каждого пользователя может быть только один бюджет
- Категория не может быть удалена, если есть связанные транзакции
- Транзакции сортируются по дате (новые сверху)
- Категории сортируются по типу и имени
- Описание транзакции необязательно
- Бюджет используется для расчёта остатка в Dashboard

## 7. Итог

Модели проекта образуют простую, но гибкую структуру:
- **Budget** отвечает за месячный лимит
- **Category** структурирует операции
- **Transaction** хранит финансовые данные

Эта архитектура обеспечивает удобную работу с бюджетом, отчётами и фильтрами.
